## 헷갈리는 것
쿠키 : 서버가 클라이언트 브라우저에 보내는 작은 텍스트 파일로, 항상 서버로 전송된다.  
세션 : 클라이언트 정보를 서버에서 관리한다. (쿠키보다 보안에 좋지만, 서버 메모리를 차지한다.)  
토큰 : 서버가 인증된 사용자에게 보내는 문자열로, 클라이언트가 요청 시 토큰을 전송해서 인증한다.  
캐시 : 자주 사용하는 데이터를 임시 저장해서 서버에 접근하지 않고 빠르게 사용할 수 있게 해준다.  
@RequestBody : HTTP 요청 메시지 바디에 있는 데이터를 직접 조회할 때 사용한다.  
@ResponseBody : HTTP 응답 메시지 바디에 return값을 넣는다.  
@RestController : @Controller에 @ResponseBody가 추가되어, 모든 메서드에 @ResponseBody가 적용된다.  
@RequestParam : HTTP 요청의 쿼리 파라미터 값을 가져올 때 사용한다.  
@ModelAttribute : HTTP 요청의 쿼리 파라미터 값이 기본 타입이 아니거나 여러 필드일 때, 사용하고 요청 파라미터를 객체로 바인딩한다.  
@PathVariable : URL 경로에서 변수 값을 추출할 때 사용한다.  

## 체크리스트 정리
엔티티 : 식별자가 필요하고, 지속해서 값을 추적, 변경해야 하는 객체다.  
값 타입 : 식별자가 없고, 생명주기를 엔티티에 의존하고, 공유하면 안 된다.  
연관관계 : 다중성에는 다대일, 일대다, 일대일, 다대다가 있고, 방향에는 단방향, 양방향이 있다.  
연관관계의 주인은 외래키가 있는 곳을 주인으로 정한다. (일대다는 일이 연관관계 주인인데 다 쪽에 외래키가 있어서 다대일 양방향 매핑을 사용하는 게 좋다.)  
지연 로딩은 프록시로 조회해서 사용하는 시점에 초기화한다.  
즉시 로딩은 프록시가 아닌 실제 엔티티를 한 번에 조회한다.  
N+1 문제는 Team을 조회할 때, Team에서 1번, Member에서 n번의 쿼리가 나오는 문제다.  
트랜잭션은 영속성 컨텍스트와 데이터베이스 커넥션을 유지시켜 지연 로딩을 할 수 있게 해준다.  
서비스 계층은 유스케이스 단위의 비즈니스 로직을 수행하고, 엔티티와 저장소를 조합해서 트랜잭션을 관리한다.
